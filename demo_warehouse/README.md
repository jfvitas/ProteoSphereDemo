# Bundled demo warehouse

Generated by `build_demo_warehouse.py` + `materialize_full_demo_warehouse.py`
+ `materialize_papers_and_audit.py`. Total size: ~158 MB.

## Contents

`catalog/v2.duckdb` — single DuckDB file (~154 MB) with these tables:

### DTI benchmarks (baseline)
- `davis_proteins` (442), `davis_ligands` (68), `davis_interactions` (30,056),
  `davis_bridge_uniprot` (1,588)
- `kiba_proteins` (229), `kiba_ligands` (2,111), `kiba_interactions` (118,253),
  `kiba_bridge_uniprot` (229)
- `gtopdb_targets` (3,369), `gtopdb_ligands` (13,727),
  `gtopdb_interactions` (23,904), `gtopdb_bridge_uniprot` (2,581)
- `pdbbind_interactions` (24,804)

### PPI benchmarks (baseline)
- `hippie_bridge_uniprot` (34,055), `huri_bridge_uniprot` (34,691),
  `s_3did_bridge_uniprot` (10,723)

### Functional / sequence / orthology memberships (baseline)
- `v2_ec_class_membership` (9,505)
- `v2_ortholog_cluster_membership` (68,987)
- `v2_sequence_cluster_membership` (71,056)

### Motif / family / structure relationships (new — Stage 19)
- `v2_motif_membership` (79,675) — Pfam + InterPro + ELM + Motivated
  Proteins family memberships for every UniProt in the warehouse
  (universe = 57,437 accessions including 11 globin/kinase external
  homologs added so the manuscript's 1MBO ↔ 1HHO globin example is
  reachable). Sample query:

  ```sql
  -- All pairs of warehouse proteins that share the globin Pfam family
  SELECT a.uniprot, b.uniprot, a.identifier, a.label
  FROM v2_motif_membership a JOIN v2_motif_membership b
    ON a.namespace='Pfam' AND a.identifier='PF00042'
   AND b.namespace='Pfam' AND b.identifier='PF00042'
   AND a.uniprot < b.uniprot;
  ```

- `v2_globin_family_members` (19) — explicit globin reference roster
  (myoglobin, hemoglobin α/β/δ/ε/γ, neuroglobin, cytoglobin,
  leghemoglobin) with canonical PDB IDs and PF00042/IPR000971 stamps.

- `v2_pdb_uniprot` (127,745) — PDB ID ↔ UniProt accession xref for
  every UniProt in the warehouse universe. Lets the Library tab join
  on PDB ID and resolve 1MBO → P02185, 1HHO → {P69905, P68871}, etc.

### Ligand chemical-similarity (new — Stage 19)
- `v2_scaffold_membership` (13,540) — Bemis-Murcko scaffold for every
  ligand in davis_ligands + kiba_ligands + gtopdb_ligands, computed
  with RDKit. Scaffolds shared by 2+ ligands are leakage-edge sources
  for DTI splits.
- `v2_scaffold_edges_summary` (1,484) — pre-aggregated per-scaffold
  counts of how many ligands carry each scaffold (multi-source flagged).

### Paper rosters (new — Stage 19)
- `papers_rows` (1,140,931) — row-level dataset rosters for every
  paper in the manuscript (58 CSVs, capped at 25,000 rows per paper +
  30 MB byte cap to keep the bundle under 250 MB). Columns:
  `paper_id`, `family`, `row_index`, `dataset`, `split`, `drug_id`,
  `drug_smiles`, `target_id`, `target_seq_sha8`, `affinity_value`,
  `pdb_id`, `protein_id`, `pair_partner_id`, `payload_json`,
  `snapshot_id`.
- `papers_metadata` (58) — per-paper provenance: family, n_rows,
  n_rows_total (true CSV count), source URLs + sha256.

### Cross-benchmark leakage audit (new — Stage 19)
- `pinder_plinder_audit` (6) — PINDER ↔ PLINDER cross-leakage verdicts
  across 6 comparisons (pinder_test↔plinder_train, plinder_test↔pinder_train,
  pinder_intra, plinder_intra etc.), composite leakage score per
  comparison.
- `pinder_plinder_axis_overlap` (42) — per-axis overlap detail:
  direct_pdb, accession_root, uniref100, uniref90, uniref50,
  pfam_family, interpro_family (ligand axes were unavailable upstream).

### Provenance
- `ingest_runs` (1) — one row per second-pass materialization (records
  which tables were produced and the snapshot_id).

## Other files

- `embeddings/` — pre-computed ESM-2 protein embeddings for every
  Davis + KIBA protein, so the first training run doesn't trigger a
  4 GB fair-esm download.
- `materialize_manifest.json` — JSON manifest from the materializer
  with stats per table and the snapshot_id.

## Reproducibility

To rebuild from scratch on a maintainer machine that has the legacy
ProteoSphere reference_library:

```bash
cd build/release-repos/proteosphere-model-studio
python build_demo_warehouse.py             # baseline (~24 MB)
# build_demo_warehouse.py invokes materialize_full_demo_warehouse.py
# (motif, scaffold, pdb_uniprot, globin family) and then
# materialize_papers_and_audit.py (papers_rows, papers_metadata,
# pinder_plinder_audit, pinder_plinder_axis_overlap).
```

The `materialize_*` scripts can also be re-run independently — each
table is `DROP`'d and recreated, so re-running is idempotent.

Total size ~158 MB. Sufficient for end-to-end CPU training of a
DeepDTA-style model on Davis warm-split *plus* the manuscript's
headline globin / kinase / PINDER-PLINDER findings reachable from
the Library tab.
